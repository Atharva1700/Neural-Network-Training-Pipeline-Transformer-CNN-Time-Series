⚡ QUICK START

make train        # Train models

make mlflow       # Start MLflow UI

make test         # Run tests

🔧 CUSTOMIZATION
═══════════════════════════════════════════════════════════════════════════════

Modify Hyperparameters:
─────────────────────
train_pipeline.py:
- num_epochs: Training duration
- batch_size: Memory vs speed tradeoff
- learning_rate: Model convergence
- lora_r, lora_alpha: LoRA configuration

Add New Models:
───────────────
1. Create model class in train_pipeline.py
2. Add training method to MLPipeline
3. Update API endpoints in api_server.py
4. Add tests to test_pipeline.py

Integrate Custom Data:
──────────────────────
Replace generate_sample_data() with your data loader:
- Load from CSV, JSON, database
- Apply preprocessing
- Split train/validation sets

🆘 TROUBLESHOOTING
═══════════════════════════════════════════════════════════════════════════════

API won't start:
→ Check port 8000 is available: lsof -i :8000
→ Verify dependencies: pip install -r requirements.txt

Models not loading:
→ Run training first: python train_pipeline.py
→ Check models directory exists: mkdir -p models

MLflow UI errors:
→ Check port 5000 is available
→ Verify mlruns directory exists

Docker issues:
→ Check Docker daemon: docker ps
→ Rebuild images: docker-compose build --no-cache

Out of memory:
→ Reduce batch_size in training scripts
→ Use CPU instead of GPU: export CUDA_VISIBLE_DEVICES=""
